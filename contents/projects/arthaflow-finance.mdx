## Overview
ArthaFlow Finance is a personal finance tracking web application built to help users manage income, expenses, wallets, budgets, and financial reports in one integrated platform. The application also includes AI-powered receipt scanning, profile customization, export tools, and multi-currency support to make daily financial management more practical and efficient.

---

## Key Features
- Record and manage **income and expense transactions**
- Organize finances across multiple **wallets**
- Track monthly spending with **budget management**
- Monitor financial activity through an **interactive dashboard**
- Scan receipts automatically using **AI-powered extraction**
- Export transaction history in **CSV** and **PDF** formats
- Customize profile and financial preferences
- Support multiple currencies:
  - Indonesian Rupiah (**IDR**)
  - US Dollar (**USD**)
  - Euro (**EUR**)
  - Singapore Dollar (**SGD**)
  - Japanese Yen (**JPY**)
  - Malaysian Ringgit (**MYR**)

## How It Works (The Tech Behind It)
To support a complete finance tracking workflow, ArthaFlow integrates several modern web technologies:

- **Vite + React.js**  
  Used to build a fast, component-based frontend with a responsive user interface.

- **TypeScript**  
  Ensures safer and more maintainable development across both frontend and backend.

- **Express.js**  
  Powers the backend API for handling transactions, wallets, budgets, settings, export, and authentication flows.

- **Better Auth**  
  Manages authentication, session handling, and protected routes.

- **PostgreSQL + Drizzle ORM**  
  Provide structured data storage with type-safe database queries and schema management.

- **Supabase Storage**  
  Handles file storage for user profile images in a serverless-friendly way.

- **Google Gemini**  
  Enables AI-based receipt scanning to extract structured transaction data from uploaded receipt images.

- **PDFKit**  
  Generates PDF financial reports with formatted transaction summaries.

- **Recharts**  
  Used to visualize spending patterns, balance trends, and expense distribution in the dashboard.

## Challenges & Improvements
Some of the main technical challenges in this project included:

- **Backend stability**  
  The API initially faced instability during startup because of database and authentication initialization. This was improved by restructuring the database layer with a lazy initialization approach and typed database access.

- **Authentication flow**  
  Session handling caused blank states and loading issues in some cases. This was fixed by refining auth flow, improving fallback handling, and stabilizing protected route registration.

- **Profile photo upload**  
  Uploading images in a serverless deployment introduced multipart and storage issues. The final solution used Multer memory storage, Supabase Storage, and public image URLs instead of base64 storage.

- **Dashboard responsiveness**  
  Chart rendering and layout responsiveness required additional refinement to make the dashboard more stable across different screen sizes.

- **Export formatting**  
  CSV and PDF exports initially displayed raw numeric values and less structured layouts. The export system was improved with currency-aware formatting and cleaner report presentation.
