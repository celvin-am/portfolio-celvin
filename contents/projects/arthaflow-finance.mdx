Overview

ArthaFlow Finance is a personal finance tracking web application designed to help users manage income, expenses, wallets, budgets, and financial reports in one integrated platform. The app also supports AI-based receipt scanning and export features for more practical daily financial management.

Tech Stack

Frontend
Vite, React.js, TypeScript, Tailwind CSS, React Router, TanStack Query, Axios, Recharts

Backend
Node.js, Express.js, TypeScript, Better Auth, Multer

Database & Services
PostgreSQL, Drizzle ORM, Supabase Storage, Google Gemini, PDFKit

Deployment
Vercel

Key Features
Income and expense transaction management
Wallet and budget tracking
Interactive financial dashboard
AI-powered receipt scanning
Profile and preference settings
CSV and PDF export
Responsive web interface
Multi-currency support: Indonesian Rupiah (IDR), US Dollar (USD), Euro (EUR), Singapore Dollar (SGD), Japanese Yen (JPY), and Malaysian Ringgit (MYR)
Challenges & Solutions

Backend stability
The API initially faced instability during startup due to database and authentication initialization. This was resolved by restructuring the database layer with a lazy initialization approach and improving typed database access.

Authentication flow
Session handling caused blank states and loading issues in some cases. The solution involved refining the auth flow, stabilizing route registration, and improving unauthenticated fallbacks on the frontend.

Profile photo upload
Uploading profile images in a serverless environment introduced issues around multipart handling and image storage. This was solved by using Multer memory storage, Supabase Storage for file hosting, and storing image URLs instead of base64 data.

Dashboard responsiveness
Several chart and layout issues appeared when rendering data on different screen sizes. These were addressed by restructuring responsive containers and controlling chart rendering based on layout readiness.

Export formatting
Transaction exports initially showed unformatted values and unstructured layouts. The export feature was improved by formatting amounts based on the user’s selected currency and redesigning the PDF into a cleaner report layout.

Lessons Learned
Strong coordination between frontend and backend is essential for a stable user experience
Type-safe architecture makes debugging and refactoring much more reliable
Serverless environments require careful handling for uploads and initialization logic
AI-powered features still need validation and cleanup to produce dependable results
Iterative debugging can lead to meaningful architectural improvements, not just bug fixes
Project Outcome

ArthaFlow evolved into a more stable and production-ready application after resolving issues across authentication, dashboard rendering, file upload, export generation, and API structure. The project strengthened both my full-stack debugging skills and my understanding of building practical financial tools.
